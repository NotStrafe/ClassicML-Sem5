# Команда и распределение ролей (3 человека)

### Member 1 — EDA, очистка, базовая подготовка данных (параллельно и стартует первым)

**Цель:** дать чистую и качественную базу, выявить проблемные места, подготовить базовые baseline данные/фичи.

**Задачи:**

* Полный EDA (структура, пропуски, типы, распределения, корреляции, target-leakage check).
* Анализ временных и географических полей (если есть): проверить временные метки, timezone, преобразовать в datetime.
* Обработка пропусков и выбросов (пояснить логику: удаление vs. imputation).
* Преобразования: one-hot / target encoding для категорий, нормализация/стандартизация числовых признаков.
* Создать train/validation/test split (стратифицированный по target; либо временной split, если дата важна).
* Выпустить минимальный baseline: простая логистическая регрессия/DecisionTree и отчёт по метрикам (ROC-AUC, Precision, Recall, F1, Confusion matrix).
* Подготовить «clean_data.csv» и notebook cell со всеми preprocessing steps (чтобы другие могли воспроизвести).

**Артефакты:**

* Notebook: `01_EDA_and_Cleaning.ipynb`
* `clean_data.csv` (или датасет в /data)
* Отчёт (markdown) с найденными проблемами и решениями


### Member 2 — Feature engineering, уникальная идея, интерпретация признаков

**Цель:** извлечь информативные признаки, добавить «уникальную» фичу, сделать анализ важности и подготовить признаки для моделей.

**Задачи:**

* Базовый feature engineering: агрегаты (rolling windows, lags — если данные временные), взаимодействия, полиномиальные признаки там, где нужно.
* Гео-фичи (если есть координаты): расстояния до ближайшего активного разлома (если источники), кластеризация (KMeans по локациям) — хотя бы кластер id как признак.
* Временные признаки: час/день/месяц, сезонность, time since last event, frequency in last T window.
* Feature selection: корреляция, variance threshold, mutual information, и L1/Tree-based selection.
* Уникальная «плюшечка» (выберите 1):

  * A) Стек «event history» — построить признаки, показывающие число событий в радиусе R и окне времени T (простая пространственно-временная агрегация).
  * B) SHAP-информированные признаки: создаёте мета-признак, который агрегирует важности SHAP-значений для группы признаков.
  * C) Multi-scale features: агрегации с разными окнами (1h, 6h, 24h) и их отношения.
* Подготовить пайплайн (sklearn Pipeline / joblib) для преобразований, чтобы другие могли подставить модели.
* Провести первичную оценку важности признаков (Permutation importance + tree feature importance + SHAP summary plots).

**Артефакты:**

* Notebook: `02_Feature_Engineering.ipynb`
* `feature_pipeline.joblib` или функция `transform_features(df)` в `features.py`
* SHAP визуализации/plots (png)
* Описание уникальной идеи и краткий тест её влияния (A/B сравнение метрик)

### Member 3 — Моделирование, тюнинг, сравнение, финальный отчет и презентация

**Цель:** обучить модели, сделать подбор гиперпараметров, провести сравнительный статистический анализ, собрать финальные результаты и подготовить слайды + видео.

**Задачи:**

* Реализовать и протестировать стандартные модели: DecisionTree, RandomForest, XGBoost/LightGBM/CatBoost, SVM (если целесообразно), ваша собственная логистическая регрессия (см. ниже).
* **Реализация собственной логистической регрессии** (обязательно для команды из 3): SGD / batch gradient descent с L2 регуляризацией, функция logloss, векторизованный numpy-код, проверка эквивалентности с sklearn.LogisticRegression (сравнить по метрикам).
* Кросс-валидация: stratified K-fold (или TimeSeriesSplit если временной), отчёты по mean±std метрики.
* Hyperparameter tuning:

  * Быстрый RandomSearch/Optuna для 1–2 моделей.
  * **Опционально (даёт +): Optuna для CatBoost** — если хватит времени, выделите 16–32 trials минимум.
* Построить ансамбли и/или stacking (meta-model) — простой blender логистическая регрессия поверх сильных моделей даст прирост.
* Сравнительный статистический анализ: таблица метрик (AUC, F1, Precision@k, Recall@k), ROC и PR curves, confusion matrices, calibration plots.
* Подготовить финальный ноутбук с выводами, рекомендации для продакшн (threshold selection, calibration).
* Подготовить презентацию (10–15 слайдов) и 5-мин видео: демонстрация уникальной фичи и краткое сравнение модели.
* Подготовить `requirements.txt` и README по запуску.

**Артефакты:**

* Notebook: `03_Modeling_and_Tuning.ipynb`
* `models/` (скрипты/сохранённые модели: e.g., `catboost_model.cbm`, `stacker.pkl`)
* `presentation.pdf` (или PowerPoint) и файл видео (`defend.mp4`)
* `README.md`, `requirements.txt`
